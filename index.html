1. Što je to CSS

CSS (Cascading Style Sheets), odnosno kaskadni stilovi, jednostavan su mehanizam za dodavanje stilova: fontova, boja razmaka između paragrafa, uređivanje tablica.

Svojim dolaskom CSS je izazvao pravu revoluciju na internetu i to zahvaljujući nizu prednosti koje ima pred tabličnim layoutom (korištenje tablica za formiranje stranice).

Korištenjem CSS-a postalo je moguće odvojiti prezentaciju podataka i dizajn od same strukture podataka. HTML kôd postaje pregledniji i manji što znači da ga je puno lakše kontrolirati, a također je moguće jednostavnom primjenom parametara promijeniti izgled stranice.

CSS je donio čitav niz načina za uređivanje prikaza podataka koji do tada nisu postojali u samom HTML-u, a web programeri su razvili korisne tehnike kojima možete uštediti dragocjeno vrijeme prilikom izrade internet stranica.

Mogučnosti formatiranja HTML-a poprilično su ograničene. Kada dizajniramo izgled stranice u HTML-u, ograničeni smo na tablice, kontrola fontova, i nekoliko stilova teksta poput bold i italic.

1.1 Kratka povijest CSS-a

CSS je prvi put predložio Håkon Wium Lie 10. listopada 1994. godine. U to je vrijeme Lie radio s Timom Berners-Leejem u CERN-u.

Otprilike u isto vrijeme predloženo je nekoliko drugih jezika listova stilova za web, a rasprave o javnim mailing listama i unutar World Wide Web Consortiuma rezultirale su objavljivanjem prve W3C CSS preporuke (CSS1) 1996. godine.

1.2 Verzije CSS-a/moduli

CSS ima različite razine i profile. Svaka razina CSS-a nadovezuje se na posljednju, obično dodajući nove značajke i obično se označava kao CSS 1, CSS 2, CSS 3 i CSS 4. Profili su obično podskup jedne ili više razina CSS-a izgrađenih za određeni uređaj ili korisničko sučelje. Trenutno postoje profili za mobilne uređaje, pisače i televizore. Profile ne treba miješati s vrstama medija, koje su dodane u CSS 2.

1.2.1 CSS1

Prva specifikacija CSS-a koja je postala službena preporuka W3C-a je CSS razina 1, objavljena 17. prosinca 1996. Håkon Wium Lie i Bert Bos zaslužni kao i izvorni programeri. Među njegovim mogućnostima su potpora za:

  • Svojstva fonta kao što su tipografija i isticanje ("emphasis")
  • Boja teksta, pozadina i drugih elemenata
  • Atributi teksta kao što su razmak između riječi, slova i redaka teksta
  • Poravnanje teksts i slika, tablica i drugih elemenata
  • Margina, obrub ("border"), podstava ("padding") i pozicioniranje za većinu elemenata
  • Jedinstvena identifikacija i generička klasifikacija skupina atributa

1.2.2 CSS2

Specifikaciju CSS razine 2 razvio je W3C i objavio kao preporuku u svibnju 1998. Superskup CSS 1, CSS 2 uključuje niz novih mogućnosti kao što su apsolutno, relativno i fiksno pozicioniranje elemenata i z-indeksa, koncept vrsta medija, podrška za zvučne listove stilova (koji su kasnije zamijenjeni CSS 3 govornim modulima) i dvosmjerni tekst te nova svojstva fonta kao što su sjene.

1.2.3 CSS3

Za razliku od CSS-a 2, koji je velika pojedinačna specifikacija koja definira različite značajke, CSS 3 je podijeljen u nekoliko zasebnih dokumenata pod nazivom "moduli". Svaki modul dodaje nove mogućnosti ili proširuje značajke definirane u CSS 2, čuvajući kompatibilnost sa starijim verzijama. Rad na razini CSS-a 3 započeo je u vrijeme objave izvorne preporuke CSS-a 2. Najraniji nacrti CSS-a 3 objavljeni su u lipnju 1999. Zbog modularizacije različiti moduli imaju različitu stabilnost i statuse

1.2.4 CSS4

Ne postoji jedinstvena, integrirana specifikacija CSS4 jer je specifikacija podijeljena na mnoge zasebne module koji se samostalno izjednačavaju. Moduli koji se temelje na stvarima iz CSS razine 2 započeli su na razini 3. Neki od njih već su dostigli razinu 4 ili se već približavaju razini 5. Ostali moduli koji definiraju potpuno nove funkcionalnosti, kao što je Flexbox, označeni su kao razina 1, a neki od njih približavaju se razini 2. Radna skupina CSS-a ponekad objavljuje "Snimke", zbirku cijelih modula i dijelova drugih skica koje se smatraju dovoljno stabilnima da ih implementiraju programeri preglednika. Do sada je pet takvih dokumenata "najbolje prakse" objavljeno kao Bilješke, 2007., 2010., 2015., 2017., i 2018.

1.2 Doba Vendor prefixesa

Pojedinačni dobavljači preglednika povremeno su uvodili nove parametre prije standardizacije i univerzalizacije. Da bi spriječili ometanje budućih implementacija, dobavljači su pripremili jedinstvena imena parametrima, kao što su -moz- za Mozilla Firefox, -webkit- nazvan po modulu za pregledavanje Apple Safarija, -o- za Opera Browser i -ms- za Microsoft Internet Explorer. Ponekad parametri s prefiksom dobavljača kao što su -moz-radijalno-gradijent i -webkit-linearni-gradijent imaju nešto drugačiju sintaksu u usporedbi s njihovim kolegama koji nisu dobavljač-prefiks. Prefiksirana svojstva postaju zastarjela do trenutka standardizacije. Programi su dostupni za automatsko dodavanje prefiksa za starije preglednike i za ukagavanje standardiziranih verzija prefiksitinih parametara. Budući da su prefiksi ograničeni na mali podskup preglednika, uklanjanje prefiksa omogućuje drugim preglednicima da vide funkcionalnost.

1.3 Doba poscss/precss plugina

Dugo vremena pisanje CSS-a bilo je dovoljno jednostavno: dodajte.css nastavak u datoteku, otvorite uređivač teksta i počnite pisati. No, tijekom godina, CSS je razvio vlastiti ekosustav tehnologija, od preprocesora kao što su Sass i Less do metodologija kao što su BEM i Atomic CSS, do novijih razvoja kao što je CSS-in-JS. Dok je BEM ispred čopora ovdje, Atomic CSS također počinje dobivati na snazi sa svojom potpuno novom filozofijom usmjerenom na komunalne klase. CSS-in-JS je jedan od najvećih događaja u posljednjih nekoliko godina u CSS svijetu. Baš kao što su preprocesori zaobišli CSS kako bi predstavili vlastiti skup novih značajki prije više od deset godina, CSS-in-JS knjižnice piggyback na JavaScriptovoj snazi da naprave vlastiti put bez čekanja da CSS sustigne.

2. Struktura i sintaksa CSS-a

Stilski obrasci sačinjeni su od stilskih pravila. Svako pravilo ima dva dijela:  Selektor: Određuje element na koji se stilsko pravilo odnosi  Deklaracija: Određuje kako izgleda sadržaj opisan CSS-om Koristimo se sa setom interpunkcijskih i posebnih znakova kako bi definirali stilsko pravilo. Sintaksa za stilska pravila uvijek slijedi sljedeći uzorak: selektor {deklaracija;} Deklaracija se dijeli na dvije stavke: 1. Properties (aspekti kako da računalo prikaže tekst i grafiku). 2. Values (podaci koji određuju kao želimo da tekst i slike izgledaju na našoj stranici). Properties se od vrijednosti (value) u deklaraciji odvaja dvotočkom, a svaka deklaracija završava s točkom-zarez: selector {property: value;}

2.1 Selektori i deklaracije

Za jedan selektor možemo napraviti nekoliko stilskih pravila, svaki s jednom deklaracijom: 


				h1 {color: teal;}
				h1 {font-family:Arial;}
				h1 {font-size: 36px;}
No, tako velika kolekcija stilsih pravila postaje teška za upotrebu. CSS vam dozvoljava da kombinirate nekoliko deklaracija u jedno stillsko pravilo koje utječe na karakteristike prikaza pojedinog selektora: Sve deklaracije za h1 selektor su unutar istih vitičastih zagrada ({}) i odvojene su točkazarezom (;). Možemo staviti koliko god hoćemo deklaracija u stilsko pravilo, samo trebamo svaku deklaraciju završiti s točka-zarezom. 


					h1 {
						color: teal; font-family: Arial; font-size: 36px; }

Možemo napraviti da se isti set deklaracija odnosi na cijelu skupinu selektora. Selektore je potrebno odvojiti zarezima. Sljedeće stilsko pravilo ostavlja deklaracije za boju teksta, font i veličinu fonta: 


					h1, h2, h3 { color: teal; font-family:
						Arial; font-size: 36px; }

2.2 Rad sa stilskim klasama i id-ovima

Selektori se odnose na sljedeće::

  • all elements of a specific type, e.g. the second-level headers h2
  • elements specified by attribute, in particular:
    • ID: identifikator jedinstven unutar dokumenta, identificiran prefiksom hasha, npr. #id
    • Klasa: identifikator koji može označiti više elemenata u dokumentu, označen prefiksom razdoblja, npr.classname
  • elements depending on how they are placed relative to others in the document tree.
